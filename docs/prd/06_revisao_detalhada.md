# PRD 06: Revisão Detalhada

## Objetivo

Permitir revisão humana produtiva, rastreável e segura dos campos extraídos.

## Componentes

1. Cabeçalho da ocorrência
2. Checklist de obrigatórios
3. Visualizador do documento
4. Campos agrupados
5. Evidência por campo
6. Candidatos alternativos
7. Edição manual
8. Aprovação de campo
9. Justificativa de ausência
10. Histórico de revisão

## Regras

1. Editar exige permissão.
2. Aprovar exige permissão.
3. Ver dado sensível exige permissão.
4. Toda edição gera Audit Log.
5. Campo manual aprovado prevalece.
6. Ocorrência não aprova com obrigatório inválido ou pendente sem justificativa.

## Critérios de aceite

1. Tela abre ocorrência.
2. Checklist mostra pendências.
3. Evidência abre em painel.
4. Campo pode ser editado.
5. Campo pode ser aprovado.
6. Aprovação da ocorrência cria snapshot.
