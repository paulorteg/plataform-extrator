# PRD 13: Backlog para Codex ou Claude Code

## Objetivo

Orientar a implementação assistida por IA com tarefas pequenas, seguras, testáveis e alinhadas à documentação.

## Regras

1. Uma issue por vez.
2. Escopo pequeno.
3. Ler AGENTS.md antes de implementar.
4. Não fazer refatorações amplas sem pedido.
5. Criar testes.
6. Validar permissões.
7. Validar organization_id.
8. Não expor dados sensíveis.

## Backlog inicial

O backlog operacional está em `docs/technical/issues_iniciais.md`.

## Critérios de aceite

1. Issues possuem contexto.
2. Issues possuem escopo.
3. Issues possuem critérios de aceite.
4. Issues possuem testes obrigatórios.
5. Issues indicam o que não fazer.
